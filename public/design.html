3 · Arquitetura de tokens — 3 camadas

global → alias → componente

Tokens fluem em três níveis: o valor bruto vive num único lugar; o resto referencia. Trocar a marca inteira = trocar a camada global; nenhuma tela precisa mudar.

CAMADA 1 · GLOBAL (primitivos)

O valor bruto

Escala completa de cor, sem contexto de uso. Definida no DESIGN.md, nunca consumida direto por telas.

--color-primary-500: #013a6a;
CAMADA 2 · ALIAS (semânticos)

A intenção de uso

Nomeia o papel, não a cor. É o que os temas (dark/light/armory/elephant) sobrescrevem.

--color-action: var(--color-primary-500);
CAMADA 3 · COMPONENTE

O uso específico

Token de componente referencia o alias. Telas consomem só esta camada (ou a alias).

.btn-primary { background: var(--color-action); }

4 · Cores das marcas + acessibilidade WCAG

—

Paletas das 4 marcas do ecossistema. Cada token mostra o contraste calculado ao vivo como cor de texto sobre as superfícies do tema dark (◐) e light (○). WCAG 2.1: AA exige 4.5:1 (texto normal) · AAA exige 7:1. Tokens reprovados nos dois fundos só podem ser usados como cor decorativa/de fundo — nunca como texto. Tokens operacionais (semantic/status + neutrals dos temas) seguem no design-tokens.css — são infraestrutura do produto, não identidade de marca.

AAA ≥ 7:1 AA ≥ 4.5:1 ✕ < 4.5:1 (não usar como texto) ◐ = sobre surface dark · ○ = sobre surface light

8 · UX Writing — tom de voz na interface

PT-BR · direto · honesto

A interface fala como o time fala: direto, técnico-acessível, sem cerimônia. E é radicalmente honesta com dados — a Regra 7 vale também pra linguagem.

Estado vazio

✅ "Nenhum alerta no momento." — diz o que aconteceu
❌ "Oops! Parece que não há nada aqui 😢" — infantiliza

Erro

✅ "Não foi possível carregar alertas. Recarregue a página." — causa + ação
❌ "Erro 500" — código sem caminho de saída

Carregamento

✅ "⏳ carregando tokens…" — específico, minúsculo
❌ "Aguarde, estamos preparando tudo para você!" — vago, prolixo

Dado sem fonte (Regra 7)

✅ "⏳ Win rate: aguarda integração Zoho stage" — gap exposto
❌ Mostrar "23%" inventado sem etiqueta — proibido

9 · Governança

Regra 8 — DESIGN.md é a fonte única

Mudança de cor, tipo, spacing, radius ou componente atômico passa pelo workflow abaixo. Sem exceção — é o que mantém 30+ telas consistentes sem retrabalho.

1 · EDITARToken no DESIGN.mdvault/00-contexto/DESIGN.md
2 · GERARRoda o geradorpython …/design/gen_tokens.py
3 · PROPAGARReload — todas as telas atualizampublic/design-tokens.css
4 · VALIDARConferir aqui no viewer/design

✅ Do

  • Consumir sempre tokens — var(--color-primary-500)
  • Linkar /design-tokens.css em toda tela nova
  • Badge de transparência em todo dado (Regra 7)
  • Dark padrão · light/armory/elephant via data-theme
  • Contraste AA mínimo (4.5:1) pra qualquer texto
  • Touch targets ≥ 44px · focus visível · PT-BR

❌ Don't

  • Hardcodar hex em CSS local — --meu-azul não existe
  • Misturar identidades (Office · ERP.ngo · GE) sem propósito
  • Fontes fora da stack (Roboto, Helvetica, etc.)
  • Override com !important — reflita no DESIGN.md
  • Número fake se passando por real — nunca, em hipótese alguma
  • Cor como único indicador de estado (use ícone + texto)
